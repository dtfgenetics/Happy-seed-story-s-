## Purpose

These are the operating rules for the Happy Seed Stories / Seed Valley publishing system.

## Core Rules

1. Story comes first. Science supports the story, not the other way around.
2. Science must be accurate and age-appropriate.
3. Use careful wording when science has exceptions: many, some, often, may, can.
4. Every book must have one main character problem, one science lesson, and one moral lesson.
5. Art must match the approved Seed Valley visual direction.
6. Every file must have an ID, version, status, and location.
7. GitHub is the master source. Google Docs are review copies. Drive stores assets. Sheets tracks status.
8. No file is final until it is marked Locked in the tracker.

## Status Flow

Idea -> Outlined -> Drafting -> In Review -> Needs Fixes -> Approved -> Locked -> Exported -> Published -> Archived
