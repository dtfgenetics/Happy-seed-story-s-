## Purpose

This file defines how versions, approvals, and locked files work.

## Version Numbers

- v0.1 = rough outline
- v0.3 = expanded outline
- v0.5 = working draft
- v0.8 = review draft
- v0.9 = approved pending final polish
- v1.0 = locked master
- v1.1 = corrected update after lock

## Lock Rule

A file is not final unless:

1. It is v1.0 or higher.
2. It is marked Locked in the tracker.
3. It exists in GitHub.
4. It has matching Drive/Docs reference links.
5. Required QC checks are complete.

## Revision Notes

Every meaningful change should explain:

- What changed
- Why it changed
- Who requested it
- Which area it affects: story, science, art, classroom, publishing, publicity, or system
