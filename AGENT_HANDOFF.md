## Purpose

This file tells any future agent, editor, designer, or automation system how to work inside this repository without breaking the structure.

## Core Rule

Do not create content randomly. Follow the source files, templates, trackers, and book IDs.

## System Roles

GitHub: source of truth.  
Google Docs: review copies.  
Google Drive: assets and exports.  
Google Sheets: tracker and status board.  

## Required Before Editing

Before editing a book file, check:

1. PROJECT_INDEX.md
2. SERIES_BIBLE.md
3. SCIENCE_RULES.md
4. CHARACTER_BIBLE.md
5. ART_BIBLE.md
6. CLASSROOM_SYSTEM.md
7. The specific book folder README

## Small Push Rule

Make small commits. Verify each file after creation or edit. If a write fails twice, skip it and return later.

## Do Not Do

- Do not overwrite locked files without revision notes.
- Do not create final art before manuscript approval.
- Do not use science claims that conflict with SCIENCE_RULES.md.
- Do not add random characters without updating CHARACTER_BIBLE.md.
- Do not create exports without updating the tracker.
