## Purpose

This file explains the repository structure.

## Root Files

Root files define series-wide systems, rules, templates, and production controls.

Examples:

- README.md
- PROJECT_INDEX.md
- SERIES_BIBLE.md
- SCIENCE_RULES.md
- ART_BIBLE.md
- CHARACTER_BIBLE.md
- CLASSROOM_SYSTEM.md
- PUBLISHING_SYSTEM.md
- PUBLICITY_SYSTEM.md

## Master Control Folder

00_master_control/ holds system-level control files such as project rules, version control, decision log, research log, risk register, and quality control.

## Book Folders

Each book has its own folder:

- 01_benny_bean_wakes_up/
- 02_penny_pea_rooty_secret/
- 03_sammy_stem_stands_tall/
- 04_lola_leaf_catches_the_light/
- 05_flora_flower_pollination_parade/
- 06_freddy_fruit_holds_a_surprise/
- 07_windy_seed_takes_a_trip/
- 08_grandma_compost_hidden_feast/

## Book Subfolders

Each active book should eventually include:

- manuscript/
- art/
- education/
- publishing/
- publicity/

## Templates Folder

templates/ holds reusable production templates.

## Rule

If a file does not clearly belong to a folder or system, define its purpose before creating it.
